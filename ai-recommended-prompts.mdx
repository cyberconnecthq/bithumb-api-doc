<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>AI Recommended Prompts</strong> API returns structured Korean analysis across all
  <strong> 17 signal types</strong> covering the Bithumb-KRW market in a single call. Each item in the
  response array represents one signal, identified by its <code>type</code> field and rendered with a
  fixed canonical Korean title (never LLM-generated).
</p>

<Note>
  Refresh: **every 30 minutes**. Dagster cron `*/30 * * * *` KST. Promotion completes ≥10 min before each
  cadence mark (i.e. by `:20` and `:50`). The Bithumb FE displays 7 questions per session (3 fixed + 4
  randomized from the 14 variable types).
</Note>

<Warning>
  Signal types missing from the latest batch are **skipped** rather than returned as `null`. The array
  preserves a fixed canonical order (Q1 → Q17 below).
</Warning>

</Note>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Per-item Response Attributes</h2>

The response is an array; each item has the same shape and represents one signal type.

<h3 style={{ marginTop: 24, marginBottom: 8 }}>Identity & metadata</h3>

<ResponseField name="type" type="string" required>
  Signal type key. One of the 17 catalogue keys above.
</ResponseField>

<ResponseField name="report_time" type="string" required>
  Snapshot wall-clock in **Asia/Seoul (KST)**. Wire format: `YYYY-MM-DD HH:MM:SS`.
  Aliased as `generated_at` in some surfaces.
</ResponseField>

<ResponseField name="uid" type="string" required>
  Document identifier. Format: `<type>_YYYYMMDD_HHMM` (KST), `HHMM` aligned to the 30-min cadence
  (`HHMM ∈ {0000, 0030, 0100, ...}`).
</ResponseField>

<h3 style={{ marginTop: 24, marginBottom: 8 }}>Headline content</h3>

<ResponseField name="title" type="string" required>
  Canonical Korean title from the catalogue above (or template-rendered for Q4/Q5). Never
  LLM-generated.
</ResponseField>

<ResponseField name="summary" type="string" required>
  Korean prose, multi-bullet markdown. ≤4–5 lines. Bullets separated by `\n- ` for FE-side splitting.
</ResponseField>

<h3 style={{ marginTop: 24, marginBottom: 8 }}>Detailed analysis</h3>

<ResponseField name="detailed_analysis" type="object" required>
  Structured Korean analysis content. The concrete key shape varies per question type — refer to the live
  response for the canonical fields returned by your `type`.
</ResponseField>

<h3 style={{ marginTop: 24, marginBottom: 8 }}>Conclusion</h3>

<ResponseField name="conclusion_neutral_opinion" type="string" required>
  **Neutral** Korean synthesis combining positive and cautionary signals.
</ResponseField>

<ResponseField name="conclusion_positive_factors" type="array of strings" required>
  Korean bullets — **positive evidence only**. No qualifications, no negative phrasing.
</ResponseField>

<ResponseField name="conclusion_cautionary_factors" type="array of strings" required>
  Korean bullets — **cautionary evidence only**. No positive phrasing leaks.
</ResponseField>

<h3 style={{ marginTop: 24, marginBottom: 8 }}>Common metadata blocks</h3>

<ResponseField name="feedback" type="object">
  Common feedback metadata block (FE-rendered).
</ResponseField>

<ResponseField name="share" type="object">
  Common share metadata block (FE-rendered).
</ResponseField>
