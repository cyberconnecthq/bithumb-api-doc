<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>AI Recommended Prompts</strong> API returns structured Korean analysis across all
  <strong> 17 signal types</strong> covering the Bithumb-KRW market in a single call. Each item in the
  response array represents one signal, identified by its <code>type</code> field.
</p>

<Note>
  Refresh: **every 30 minutes** (per SRS §3.1). Promotion completes ≥10 min before each cadence mark
  (i.e. by `:20` and `:50`). The Bithumb FE displays 7 questions per session (3 fixed + 4 randomized
  from the 14 variable types).
</Note>

<Warning>
  Signal types missing from the latest batch are **skipped** rather than returned as `null`.
</Warning>

---

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Signal types (17)</h2>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Response Attributes</h2>

The endpoint returns the standard envelope `{ success, message, data }`. The `data` field is an array; each
item has the same shape (field order below matches the wire sample):

<ResponseField name="type" type="string" required>
  Signal type key. One of the 17 catalogue keys above.
</ResponseField>

<ResponseField name="conclusion" type="object" required>
  <Expandable title="properties">
    <ResponseField name="comprehensive_conclusion" type="string" required>
      Korean prose synthesizing the signal's bottom-line takeaway.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="detailed_analysis" type="object" required>
  <Expandable title="properties">
    <ResponseField name="key_factors" type="string" required>
      Korean prose describing the data points and indicators driving this signal.
    </ResponseField>
    <ResponseField name="market_direction" type="string" required>
      Korean directional label. Wire sample shows values like `"상승"`, `"하락"`, `"중립"`, `"상승세"`,
      `"상승 가능"`, `"공급 감소"`, `"활동 증가"`, `"고변동"`, `"관심 증가"`, `"이슈 주도"`,
      `"기관 수급 확인"`, `"긍정"`, `"변동성 확대"`. Open-ended phrase set, not a closed enum.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="generated_at" type="string" required>
  Snapshot wall-clock with seconds, KST. Format: `YYYY-MM-DD HH:MM:SS`.
</ResponseField>

<ResponseField name="report_time" type="string" required>
  Reference time, KST. Wire sample uses **dot-separated** date format: `YYYY.MM.DD HH:MM` (no seconds).
</ResponseField>

<ResponseField name="summary" type="string" required>
  Short Korean prose summary of the signal (~1–2 sentences).
</ResponseField>

<ResponseField name="title" type="string" required>
  LLM-generated short Korean label for this signal item (e.g. `"저평가 자산 후보"`, `"강세 자산 후보"`,
  `"BTC ETF 플로우"`). **Distinct from the canonical "suggested question" titles** — this is the FE
  card heading.
</ResponseField>

<ResponseField name="uid" type="string" required>
  Document identifier. Format: `<type>_YYYYMMDD_HHMM` (KST), `HHMM` aligned to the 30-min cadence.
</ResponseField>

<RequestExample>

```bash cURL
curl --request GET \
  --url "https://api.asksurf.ai/surf-ai/v1/market/ai-recommended-prompts" \
  --header "Authorization: <api-key>"
```

```python Python
import requests

response = requests.get(
    "https://api.asksurf.ai/surf-ai/v1/market/ai-recommended-prompts",
    headers={"Authorization": "<api-key>"},
)
prompts = response.json()
```

```javascript JavaScript
const response = await fetch(
  "https://api.asksurf.ai/surf-ai/v1/market/ai-recommended-prompts",
  { headers: { Authorization: "<api-key>" } },
);
const prompts = await response.json();
```
