title: Token Metrics
description: 30-day daily on-chain metric series for a Bithumb-listed asset, with a Korean trend summary.
openapi: openapi.json GET /v1/market/token-metrics
---

<div style={{ maxWidth: 760 }}>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>Token Metrics</strong> API returns a single coalesced on-chain metric series spanning the last
  30 days for a given <code>symbol</code>, plus a Korean trend summary. The metric is selected per asset
  based on data availability — the first available metric in the priority order below is returned, never
  with all-zero values.
</p>

<Note>
  The series is **always exactly 30 entries** covering `[generated_at - 30d, generated_at]`.
</Note>

<h3 style={{ marginTop: 20 }}>Metric coalesce order</h3>

The pipeline tries each metric in priority order; the first available, non-zero series is returned:

`active_addresses_daily` → `transaction_count` → `tokenholders` → `transaction_volume` → `turnover_rate` (fallback).

<Warning>
  All-zero series are rejected outright (e.g. the legacy HOOK regression). If every priority option returns
  zeros, the response falls back to the cached `degraded` payload.
</Warning>

---

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Response Attributes</h2>

<ResponseField name="symbol" type="string" required>
  Bithumb exchange symbol echoed back from the request (uppercase).
</ResponseField>

<ResponseField name="generated_at" type="string" required>
  Snapshot wall-clock used as the series anchor (KST ISO 8601). The 30-entry window is
  `[generated_at − 30d, generated_at]`.
</ResponseField>

<ResponseField name="analysis" type="string" required>
  Short Korean trend summary (~2–3 sentences) describing the 30-day series.
</ResponseField>

<ResponseField name="metrics" type="array of objects" required>
  Always exactly 30 items, oldest to newest. Each item:
  <Expandable title="properties">
    <ResponseField name="metric_id" type="string" required>
      Identifier of the selected metric. One of:
      `active_addresses_daily`, `transaction_count`, `tokenholders`, `transaction_volume`, `turnover_rate`.
      Same value across all 30 entries.
    </ResponseField>
    <ResponseField name="timestamp" type="string" required>
      Day-aligned wall-clock for the data point.
    </ResponseField>
    <ResponseField name="value" type="number" required>
      Numeric metric value for that day.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash cURL
curl --request GET \
  --url "https://api.asksurf.ai/surf-ai/v1/market/token-metrics?symbol=BTC" \
  --header "Authorization: <api-key>"
```

```python Python
import requests

response = requests.get(
    "https://api.asksurf.ai/surf-ai/v1/market/token-metrics",
    params={"symbol": "BTC"},
    headers={"Authorization": "<api-key>"},
)
metrics = response.json()
```

```javascript JavaScript
const response = await fetch(
  "https://api.asksurf.ai/surf-ai/v1/market/token-metrics?symbol=BTC",
  { headers: { Authorization: "<api-key>" } },
);
const metrics = await response.json();
```
