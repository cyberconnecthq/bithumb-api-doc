---
title: Token Metrics
description: 30-day daily on-chain metric series for a Bithumb-listed asset, with a Korean trend summary.
api: 'GET https://api.asksurf.ai/surf-ai/v1/market/token-metrics'
---

<div style={{ maxWidth: 760 }}>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>Token Metrics</strong> API returns a single coalesced on-chain metric series spanning the last
  30 days for a given <code>symbol</code>, plus a Korean trend summary. The metric is selected per asset
  based on data availability — the first available metric in the priority order below is returned, never
  with all-zero values.
</p>

<Note>
  The series is **always exactly 30 entries** (one per day) covering `[generated_at - 30d, generated_at]`.
  When upstream data is unavailable, a last-known-good cached payload is returned with `analysis = ""`
  rather than propagating an upstream error.
</Note>

<h3 style={{ marginTop: 20 }}>Query parameters</h3>

<ParamField query="symbol" type="string" required>
  Bithumb exchange symbol (uppercase). Example: `BTC`, `ETH`, `SOL`.
  Returns `404` if the symbol is not in the current Top-200 universe.
</ParamField>

<h3 style={{ marginTop: 20 }}>Metric coalesce order</h3>

The pipeline tries each metric in order; the first available, non-zero series is returned.

| Priority | `metric_id` | Korean | What it measures |
|---|---|---|---|
| 1 | `active_addresses_daily` | 일일 활성 주소 | Unique addresses transacting per day |
| 2 | `transaction_count` | 거래 건수 | Number of on-chain transactions per day |
| 3 | `tokenholders` | 토큰 보유자 수 | Distinct holding addresses |
| 4 | `transaction_volume` | 거래량 | On-chain transferred value |
| 5 *(fallback)* | `turnover_rate` | 회전율 | `transaction_volume / circulating supply` |

<Warning>
  All-zero series are rejected outright (e.g. the legacy HOOK regression). If every priority option returns
  zeros, the response falls back to the cached `degraded` payload.
</Warning>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Response Attributes</h2>

<ResponseField name="symbol" type="string" required>
  Bithumb exchange symbol echoed back from the request (uppercase).
</ResponseField>

<ResponseField name="analysis" type="string" required>
  Short Korean trend summary (~2–3 sentences) describing start → peak → end of the 30-day series, with a
  recent-slope note. **Empty string** when the series is too short to analyse or when the response is
  degraded (no fresh data).

  Example: `"월초 X에서 4월 중순 Y 고점 후 5월 초 Z으로 둔화. 최근 5일 +1.2% 회복."`
</ResponseField>

<ResponseField name="metrics" type="array of objects" required>
  **Always exactly 30 items**, one per day, oldest to newest. Each item:
  <Expandable title="properties">
    <ResponseField name="metric_id" type="string" required>
      Identifier of the selected metric. One of:
      `active_addresses_daily`, `transaction_count`, `tokenholders`, `transaction_volume`, `turnover_rate`.
      Same value across all 30 entries (single-metric series).
    </ResponseField>
    <ResponseField name="chain" type="string" required>
      Chain identifier the metric is sourced from (Korean rendering applied where available).
      Example: `"솔라나"`, `"이더리움"`, `"트론"`.
    </ResponseField>
    <ResponseField name="timestamp" type="string" required>
      Day-aligned wall-clock in **Asia/Seoul (KST)**. Wire format: `YYYY-MM-DD HH:MM:SS`. Daily series
      timestamps land at `00:00:00` KST.
    </ResponseField>
    <ResponseField name="value" type="number" required>
      Numeric metric value for that day. Units depend on `metric_id`
      (counts for `transaction_count` / `active_addresses_daily` / `tokenholders`,
      currency value for `transaction_volume`, ratio for `turnover_rate`).
    </ResponseField>
  </Expandable>
</ResponseField>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Errors</h2>

| Status | Code | When |
|---|---|---|
| 400 | `INVALID_PARAMS` | `symbol` is missing or empty. |
| 404 | `NOT_FOUND` | `symbol` is not in the current Top-200 universe. |

</div>

<RequestExample>

```bash cURL
curl --request GET \
  --url "https://api.asksurf.ai/surf-ai/v1/market/token-metrics?symbol=BTC" \
  --header "Authorization: <api-key>"
```

```python Python
import requests

response = requests.get(
    "https://api.asksurf.ai/surf-ai/v1/market/token-metrics",
    params={"symbol": "BTC"},
    headers={"Authorization": "<api-key>"},
)
metrics = response.json()
```

```javascript JavaScript
const response = await fetch(
  "https://api.asksurf.ai/surf-ai/v1/market/token-metrics?symbol=BTC",
  { headers: { Authorization: "<api-key>" } },
);
const metrics = await response.json();
```

</RequestExample>
