<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>AI Report</strong> API generates a structured Korean research report for each in-scope asset,
  combining a real-time market snapshot, technical indicators, sector-aware on-chain metrics, community
  sentiment, and a neutral conclusion. Coverage is limited to the <strong>top 200 Bithumb-KRW assets</strong> by market cap.
</p>

<Note>
  This report is a <strong>point-in-time snapshot</strong> intended for informational purposes only — it does
  not provide financial advice.
</Note>

<h3 style={{ marginTop: 20 }}>Cadence & freshness</h3>

<CardGroup cols={2}>
  <Card title="Narrative content" icon="clock">
    Refreshes every **4 hours** at KST `00 / 04 / 08 / 12 / 16 / 20`. Promotion to public read tables completes **≥10 minutes** before each cadence mark.
  </Card>
  <Card title="On-chain figures" icon="database">
    Refresh **daily**. Market-cap / FDV / supply / 24h volume / TVL reflect a single data snapshot taken at `generated_at`.
  </Card>
</CardGroup>

<Warning>
  If certain data is unavailable or not applicable, corresponding fields may be <code>null</code>, an empty
  string, or omitted from the response.
</Warning>

---

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Response Attributes</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The single-report object below (<code>AIReportItem</code>) is returned directly when <code>ticker</code> is
  provided, and as items in <code>reports[]</code> when it is omitted.
</p>

<ResponseField name="ticker" type="string" required>
  Bithumb exchange symbol, uppercase.
</ResponseField>

<ResponseField name="bithumb_rank" type="integer" required>
  Rank within the Bithumb-KRW Top-200 universe at `generated_at`. Range: `1`–`200`.
</ResponseField>

<ResponseField name="uid" type="string" required>
  Document identifier. Format: `ai_report_<SYMBOL>_YYYYMMDD_HHMM` (KST), with `HHMM` aligned to the
  4-hour cadence (`0000 / 0400 / 0800 / 1200 / 1600 / 2000`). Unique per `(asset, generated_at)` pair.

  Example: `ai_report_BTC_20260507_1600`.
</ResponseField>

<ResponseField name="generated_at" type="string" required>
  Snapshot wall-clock in **Asia/Seoul (KST)**. Wire format: `YYYY-MM-DD HH:MM:SS` (space-separated, no
  timezone suffix). Example: `2026-05-07 17:31:06`.

  <Note>
    The wall-clock minute/second reflects the actual write time and does **not** match the cadence mark — use
    `uid` (which encodes `HHMM`) when joining against the 4-hour batch.
  </Note>
</ResponseField>

<ResponseField name="title.title" type="string" required>
  Korean headline derived from `summary.summary`. Length: **13–16 Korean characters**
  (validator: `validate_title_length_range(13, 16)`). Generic boilerplate such as
  `"비트코인(BTC) 종합 분석"` is rejected; the title must capture the actual narrative.

  Example: `"비트코인 골든크로스 진행 매수세 확대"` (16자).


</ResponseField>

<ResponseField name="summary.summary" type="string" required>
  Multi-bullet Korean markdown. Each bullet uses `\n- 헤드라인: 본문` form and synthesizes per-metric
  scores plus sentiment into prose. The first reference to an asset includes the Korean name and
  bracketed symbol (e.g. `**비트코인(BTC)**`).
</ResponseField>

<ResponseField name="technical" type="object" required>
  Key chart / momentum interpretations across four fixed indicator slots.
  <Expandable title="properties">
    <ResponseField name="indicators" type="array of objects" required>
      **Always exactly 4 items in fixed order.** Dynamic per-asset selection is forbidden — the renderer
      cannot reorder.

      | Slot | `name` | Indicator | Parameters |
      |---|---|---|---|
      | ① 거래량 | `거래량 - OBV` | On-Balance Volume | — |
      | ② 변동성 | `변동성 - 볼린저 밴드` | Bollinger Bands | (20, 2) |
      | ③ 추세 | `추세 - MACD` | MACD | (12, 26, 9) |
      | ④ 모멘텀 | `모멘텀 - RSI` | Relative Strength Index | (14) |

      <Expandable title="item properties">
        <ResponseField name="name" type="string" required>
          One of the four fixed Korean labels above. Order is locked by slot index.
        </ResponseField>
        <ResponseField name="indicator_status" type="string" required>
          Declarative numeric snapshot (~1 sentence) in Korean. **Cites the figures.**
          Example: `"OBV 기울기 -0.0989로 하락세 지속. 최근 24시간 +58.51% 반등에도 전체 기울기는 음수 유지."`
        </ResponseField>
        <ResponseField name="analysis" type="string" required>
          Interpretive narrative (~1–2 sentences) in Korean. **Must not restate any number that already
          appears in `indicator_status`.**
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="summary" type="string" required>
      Multi-bullet Korean synthesis across the four indicators, organized as
      `* **현재 추세** / * **주요 가격대** / * **향후 관건**`.

    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="onchain" type="object" required>
  Sector-aware on-chain metrics, projection, and cautionary notes.
  <Expandable title="properties">
    <ResponseField name="sector_group" type="object" required>
      Single sector classification (one sector per asset, sourced from `kor_tags1`).
      <Expandable title="properties">
        <ResponseField name="name" type="string" required>
          Korean display label of one of the **closed 14-sector enum**:
          `AI · DeFi · DePIN · Fan Token · Gaming · Layer1 · Layer2 · Meme · Metaverse · NFT · Payments · RWA · Service · Social`.

          Wire renders Korean (e.g. `"레이어1"`, `"디파이"`). Anything outside this set is rejected.
        </ResponseField>
        <ResponseField name="description" type="string" required>
          One-line Korean description of the sector (e.g. `"기반 블록체인 네트워크"`).
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="comprehensive_data" type="array of objects" required>
      Sector-aware on-chain metric blocks. **Keys are dynamic per token**. Examples: `market_valuation`, `network_health`, `infrastructure_stability`, `development_ecosystem`.
      <Expandable title="item properties">
        <ResponseField name="title_ko" type="string" required>
          Korean section heading rendered by the front-end. Examples: `"시장 가치 평가"`,
          `"네트워크 활용 현황"`, `"개발자 활동"`, `"인프라 안정성"`.
        </ResponseField>
        <ResponseField name="content" type="string" required>
          Korean prose for that block. Numeric values use thousand separators and 만/억/조 magnitude
          abbreviations (e.g. `"시가총액 2,366조 5,657억 원..."`).
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="precautions" type="array of strings" required>
      Korean cautionary bullets surfaced from the on-chain analysis.

      Example: `"거래량이 7일 평균 0.6배 수준으로 낮아 단기 가격 변동성 확대 가능성 존재."`
    </ResponseField>

    <ResponseField name="outlook" type="string" required>
      Korean forward-looking note (1–3 sentences) interpreting the on-chain trajectory.
    </ResponseField>

    <ResponseField name="summary" type="string" required>
      Korean prose summary of the on-chain section.

    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="community" type="object" required>
  Community sentiment signals derived from KOL / social activity.
  <Expandable title="properties">
    <ResponseField name="overall_sentiment" type="object" required>
      <Expandable title="properties">
        <ResponseField name="score" type="integer" required>
          Sentiment score. Range: `1`–`100`.
        </ResponseField>
        <ResponseField name="grade" type="string" required>
          Korean grade band, deterministically derived from `score`:

          | `score` range | `grade` |
          |---|---|
          | 1–20  | 매우 부정 |
          | 21–40 | 부정 |
          | 41–60 | 중립 |
          | 61–80 | 긍정 |
          | 81–100 | 매우 긍정 |

          The unspaced variants `매우부정` / `매우긍정` are tolerated on read; canonical pipeline output
          stays spaced.
        </ResponseField>

        <ResponseField name="analysis" type="string" required>
          One Korean sentence citing at least one `key_topic` and explaining *why* the score landed
          there. Pure tonal restatement of the label alone is rejected.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="key_topics" type="array of strings" required>
      Up to **3** Korean topic strings driving the sentiment score. Every entry must
      (a) influence the sentiment score and (b) align with `overall_sentiment.grade`.

      Example from BTC sample: `"가격 목표 논의 — 90K 달성 가능성에 대한 트레이더 의견 활발 (심리 영향: +12점)"`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="conclusion" type="object" required>
  Neutral wrap-up — three required keys, format-locked.
  <Expandable title="properties">
    <ResponseField name="positive_factors" type="array of strings" required>
      Korean bullets identifying positive findings. Each bullet must follow `<현상>: <기회 의미>` form.
      Listing a number without explaining its meaning is rejected.
    </ResponseField>
    <ResponseField name="cautionary_factors" type="array of strings" required>
      Korean bullets identifying negative situations or areas requiring caution. Same `<현상>: <리스크 의미>`
      reasoning constraint as `positive_factors`.
    </ResponseField>
    <ResponseField name="overall_opinion" type="string" required>
      **Neutral** Korean synthesis paraphrasing **at least one** positive factor and **at least one**
      cautionary factor — without re-using the underlying numbers.
    </ResponseField>
  </Expandable>
</ResponseField>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Cross-cutting rules</h2>

<AccordionGroup>
  <Accordion title="Language, tone, and currency">
    - All user-facing strings are **Korean**. English abbreviations remain only when no canonical Korean
      equivalent exists (`SEC`, `CMC`, `FOMC`, `Saylor`).
    - Sentence endings use **음/슴체** (`~함`, `~판단됨`, `~예상됨`). Banned endings: `입니다`, `습니다`,
      `합니다`, `해요`, `있습니다`, `~다.`, `~입니다.`.
    - Currency is **KRW (`원`)**. USD is forbidden in user-facing fields. Magnitudes use `만 / 억 / 조`
      (e.g. `1억 1,954만 원`, `2,366조 5,657억 원`); 8+ digit raw `원` strings are rejected.
    - Timestamps are **Asia/Seoul (KST)**. UTC is forbidden in user-facing fields.
  </Accordion>

  <Accordion title="Wire format">
    - Plain JSON. Korean prose lives inside string fields; the front-end renders Markdown / line breaks /
      emphasis.
    - Where a string field's content is a list, bullets are separated by `\n- `.
    - Numeric values displayed to the user always carry thousand separators.
  </Accordion>

  <Accordion title="Substitution-bug guardrail">
    The pipeline does **not** perform regex `symbol → Korean-name` substitution on narrative text.
    The legacy bug `text.replace("BB", "바운스빗")` clobbered the Bollinger Band abbreviation. Korean asset
    names are appended in parentheses by the LLM at generation time; abbreviation localization happens via a
    glossary keyed on full tokens (`BB_Percent`, not bare `BB`).
  </Accordion>
</AccordionGroup>

<RequestExample>

```bash cURL
curl --request GET \
  --url "https://api.asksurf.ai/surf-ai/v1/market/ai-report?ticker=BTC" \
  --header "Authorization: <api-key>"
```

```python Python
import requests

response = requests.get(
    "https://api.asksurf.ai/surf-ai/v1/market/ai-report",
    params={"ticker": "BTC"},
    headers={"Authorization": "<api-key>"},
)
report = response.json()
```

```javascript JavaScript
const response = await fetch(
  "https://api.asksurf.ai/surf-ai/v1/market/ai-report?ticker=BTC",
  { headers: { Authorization: "<api-key>" } },
);
const report = await response.json();
```
