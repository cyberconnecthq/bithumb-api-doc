title: AI Recommended Assets
description: Data-driven ranking + AI interpretation for top-200 Bithumb KRW market assets.
openapi: openapi.json GET /v1/market/ai-recommended-assets
---

<div style={{ maxWidth: 760 }}>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>AI Asset Recommendation</strong> API delivers ranked insights for crypto assets listed in the
  <strong>Bithumb KRW</strong> market. It returns the <strong>top 200 tokens by market cap</strong>,
  combining multi-dimensional scoring with AI-generated interpretation.
</p>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The response is a single object keyed by all four shapes (<code>score_ranking</code>, <code>summary</code>, <code>comprehensive_evaluation</code>, <code>sentiment_trend</code>) — described below.
</p>

<Note>
  This endpoint returns a point-in-time snapshot of the most recent batch.
</Note>

<h3 style={{ marginTop: 20 }}>Variants</h3>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  Two additional endpoints return subsets of the same data:
</p>

<ul style={{ fontSize: 15, lineHeight: 1.8 }}>
  <li><code>/v1/market/ai-buy-recommendation</code> — buy recommendation (rank 1–50, score &ge; 6.0, highest first)</li>
  <li><code>/v1/market/ai-sell-recommendation</code> — sell recommendation (rank 151–200, score &lt; 6.0, lowest first)</li>
</ul>

<h3 style={{ marginTop: 20 }}>Response shapes</h3>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The response object contains four sibling keys, each holding the same <code>{`{ uid, generated_at, assets: [...] }`}</code> envelope but with a different per-asset shape:
</p>

<ul style={{ fontSize: 15, lineHeight: 1.8 }}>
  <li>
    <strong>score_ranking</strong>: asset info + <code>total_score</code>, <code>score_by_category</code>,
    <code>overall_rank</code>, <code>ranking_by_category</code>, <code>opinion</code>, and <code>interpretation</code>.
  </li>
  <li>
    <strong>summary</strong>: asset info + <code>total_score</code>, <code>score_by_category</code>,
    <code>title</code> (concise Korean label, max 20 characters), and <code>summary</code> (detailed multi-line markdown analysis interpreting scores).
  </li>
  <li>
    <strong>comprehensive_evaluation</strong>: asset info + <code>total_score</code>, <code>score_by_category</code>,
    <code>overall_rank</code>, <code>opinion</code>, and <code>interpretation</code>.
  </li>
  <li>
    <strong>sentiment_trend</strong>: asset info + <code>bullish</code>, <code>bearish</code> (integer percentages, 0–100, sum = 100),
    and <code>sentiment_label</code> (Korean label derived from bullish ratio). Filtered to tokens with sentiment data; sorted bullish-percentage descending.
  </li>
</ul>

<h3 style={{ marginTop: 20 }}>Scoring model</h3>

<div style={{ fontSize: 15, lineHeight: 1.8 }}>
  <p style={{ marginBottom: 8 }}>
    Total Score is provided together with six category scores (each 0-10):
  </p>

  <ul>
    <li><strong>Upside Energy</strong></li>
    <li><strong>Trading Activity</strong></li>
    <li><strong>Search Trend</strong></li>
    <li><strong>Price Stability</strong></li>
    <li><strong>Market Sentiment</strong></li>
    <li><strong>Price Resilience</strong></li>
  </ul>
</div>

<Warning>
  If certain data is unavailable, corresponding fields may be <code>null</code> or omitted.
</Warning>
