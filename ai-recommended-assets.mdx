<div style={{ maxWidth: 760 }}>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>AI Recommended Assets</strong> API delivers ranked insights for the
  <strong> top 200 Bithumb-KRW assets</strong> by market cap, combining a six-dimensional scoring model
  with AI-generated Korean interpretation. The response shape depends on the <code>type</code> you request.
</p>

<Note>
  Two convenience variants return subsets of the same hourly snapshot:
  [`/v1/market/ai-buy-recommendation`](/ai-buy-recommendation) (rank with `total_score ≥ 6.0`, descending)
  and [`/v1/market/ai-sell-recommendation`](/ai-sell-recommendation) (`total_score < 4.0`, ascending).
</Note>

<h3 style={{ marginTop: 20 }}>Query parameters</h3>

<ParamField query="type" type="string" required>
  Response mode. One of:
  - `score_ranking` — full ranking + per-category ranks + interpretation
  - `summary` — score + Korean title (≤20 chars) + multi-bullet markdown summary
  - `comprehensive_evaluation` — score + opinion band + interpretation
  - `sentiment_trend` — bullish/bearish percentages + Korean sentiment label
</ParamField>

<ParamField query="tickers" type="string">
  Comma-separated symbols (e.g. `BTC,ETH,SOL`). When omitted, returns all 200 assets sorted by `total_score`
  (descending). Symbols outside the Top-200 universe are silently dropped.
</ParamField>

<ParamField query="date" type="string">
  Snapshot date in `YYYY-MM-DD` (KST). The backend treats the date as KST 00:00 of that day.
  When omitted, the most recent snapshot is returned.
</ParamField>

<h3 style={{ marginTop: 20 }}>Cadence & freshness</h3>

<CardGroup cols={2}>
  <Card title="Refresh" icon="clock">
    Hourly snapshot at KST `:00`. Promotion to public read tables completes by `:50` of the previous hour.
  </Card>
  <Card title="Snapshot consistency" icon="layer-group">
    Buy / Sell / Score Ranking are three views over the **same** hourly batch. They share `generated_at`.
  </Card>
</CardGroup>

<h3 style={{ marginTop: 20 }}>The six-metric scoring model</h3>

Total Score = `Round(Average(6 metric scores), 1)`, range `0.0`–`10.0`. Each category is also `0.0`–`10.0` to one decimal place.

| Wire key | Korean (rendered) | English | Definition |
|---|---|---|---|
| `upside_energy` | 상승에너지 | Bullish Momentum | Price momentum |
| `trading_activity` | 거래활성도 | Trading Activity | Total trading volume |
| `search_trend` | 검색트렌드 | Search Trends | Public interest (search volume) |
| `price_stability` | 가격안정성 | Price Stability | Inverse of volatility (higher = more stable) |
| `market_sentiment` | 시장심리 | Market Sentiment | News + community + social media |
| `price_resilience` | 가격방어력 | Price Resilience | How well price holds support against drops |

<Warning>
  All score fields render with **exactly one decimal place** including trailing zero — `7.0`, `10.0`, `3.0`, never `7` / `10` / `3`. The Mintlify viewer may drop the trailing `.0` cosmetically; the wire bytes are correct.
</Warning>

<h3 style={{ marginTop: 20 }}>Score-to-opinion bands</h3>

`opinion` (and the band shown to users) is deterministically derived from `total_score`:

| `total_score` | `opinion` (Korean) | English |
|---|---|---|
| 0.0 – 1.9 | 매우 약세 | Very Weak |
| 2.0 – 3.9 | 약세 | Weak |
| 4.0 – 5.9 | 중립 | Neutral |
| 6.0 – 7.9 | 강세 | Strong |
| 8.0 – 10.0 | 매우 강세 | Excellent |

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Response Attributes</h2>

The top-level envelope is the same for all `type` values. The fields present on each `assets[]` item depend on `type` — see the per-mode tables below the common section.

<ResponseField name="generated_at" type="string" required>
  Snapshot wall-clock in **Asia/Seoul (KST)**. Wire format: `YYYY-MM-DD HH:MM:SS` (no timezone suffix).
</ResponseField>

<ResponseField name="uid" type="string" required>
  Batch identifier. Format: `<endpoint>_YYYYMMDD_HHMM` (KST), `HHMM` aligned to the hourly cadence
  (`HH00`). Example: `ai_recommended_assets_20260507_1700`.
</ResponseField>

<ResponseField name="assets" type="array of objects" required>
  One entry per in-scope asset. Common fields below; mode-specific fields after.

  <Expandable title="common item properties">
    <ResponseField name="symbol" type="string" required>
      Bithumb exchange symbol (uppercase). Example: `BTC`, `XRP`.
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Korean asset name. Example: `"비트코인"`, `"엑스알피"`.
    </ResponseField>
    <ResponseField name="image_url" type="string" required>
      Asset icon URL.
    </ResponseField>
    <ResponseField name="uid" type="string" required>
      Per-asset document ID. Format: `<endpoint>_<SYMBOL>_YYYYMMDD_HHMM` (KST).
    </ResponseField>
    <ResponseField name="total_score" type="number" required>
      Overall score `0.0`–`10.0`, one decimal place. `Round(Average(6 metric scores), 1)`.
    </ResponseField>
    <ResponseField name="score_by_category" type="object" required>
      Object with the six number keys `upside_energy / trading_activity / search_trend / price_stability /
      market_sentiment / price_resilience`. Each `0.0`–`10.0`, one decimal place. Korean labels and
      definitions are in the **scoring model** table above.
    </ResponseField>
  </Expandable>
</ResponseField>

<h3 style={{ marginTop: 24 }}>Mode-specific fields on each `assets[]` item</h3>

<Tabs>
  <Tab title="score_ranking">
    <ResponseField name="overall_rank" type="integer" required>
      Asset's rank by `total_score` within the universe at `generated_at`. Range `1`–`200`.
    </ResponseField>
    <ResponseField name="ranking_by_category" type="object" required>
      Object with the same six keys as `score_by_category`. Each value is an integer `1`–`200` representing
      that asset's rank within the universe for that category.
    </ResponseField>
    <ResponseField name="opinion" type="string" required>
      Korean opinion band derived from `total_score`. One of `매우 약세 / 약세 / 중립 / 강세 / 매우 강세`.
    </ResponseField>
    <ResponseField name="interpretation" type="object" required>
      Object with the same six keys as `score_by_category`. Each value is a ~1-sentence Korean explanation
      (음/슴체) citing the data behind that category's score.
    </ResponseField>
  </Tab>
  <Tab title="summary">
    <ResponseField name="title" type="string" required>
      Korean headline derived from the `summary` body. **Max 20 Korean characters.** Generic boilerplate
      (e.g. `"검색흐름 양호, 전반 우세"`) is rejected; titles must capture the actual narrative.
    </ResponseField>
    <ResponseField name="summary" type="string" required>
      Multi-bullet Korean markdown synthesizing the 6 metric scores plus sentiment trend.
      Each bullet uses `\n- 헤드라인: 본문` form (per QA `ai_assets_1` § 1.5). The first reference to an
      asset includes the Korean name and bracketed symbol.
    </ResponseField>
  </Tab>
  <Tab title="comprehensive_evaluation">
    Returns the common fields above. The exact narrative shape for this mode is not yet locked in the
    public contract — refer to the live response for the canonical structure returned by your call.
  </Tab>
  <Tab title="sentiment_trend">
    <ResponseField name="bullish" type="integer" required>
      Bullish percentage. Integer `0`–`100`.
    </ResponseField>
    <ResponseField name="bearish" type="integer" required>
      Bearish percentage. Integer `0`–`100`. **Invariant: `bullish + bearish = 100`.**
    </ResponseField>
    <ResponseField name="sentiment_label" type="string" required>
      Korean keyword label derived from the bullish ratio. One of
      `매우 부정 / 부정 / 중립 / 긍정 / 매우 긍정`.
      The unspaced variants `매우부정` / `매우긍정` are tolerated on read; canonical output stays spaced.

      <Note>
        This sentiment surface is **distinct** from AI Report's
        [`community.overall_sentiment`](/ai-report#community), which uses a `{ score: 1–100, label }`
        struct. Don't unify the two.
      </Note>
    </ResponseField>
  </Tab>
</Tabs>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Errors</h2>

The endpoint returns a `BaseResponse` envelope (`success`, `error_code`, `message`) for non-2xx responses.

| Status | Code | When |
|---|---|---|
| <span style={{color:"#d97706",fontWeight:600}}>● 400</span> | `INVALID_PARAMS` | `type` is missing or not one of the four allowed values; `date` malformed; `tickers` includes invalid characters. |
| <span style={{color:"#d97706",fontWeight:600}}>● 404</span> | `NOT_FOUND` | No snapshot exists for `date`. |

</div>

<RequestExample>

```bash cURL
curl --request GET \
  --url "https://api.asksurf.ai/surf-ai/v1/market/ai-recommended-assets?type=score_ranking&tickers=BTC,ETH" \
  --header "Authorization: <api-key>"
```

```python Python
import requests

response = requests.get(
    "https://api.asksurf.ai/surf-ai/v1/market/ai-recommended-assets",
    params={"type": "score_ranking", "tickers": "BTC,ETH"},
    headers={"Authorization": "<api-key>"},
)
data = response.json()
```

```javascript JavaScript
const params = new URLSearchParams({ type: "score_ranking", tickers: "BTC,ETH" });
const response = await fetch(
  `https://api.asksurf.ai/surf-ai/v1/market/ai-recommended-assets?${params}`,
  { headers: { Authorization: "<api-key>" } },
);
const data = await response.json();
```
