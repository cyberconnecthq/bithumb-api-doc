<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>AI Recommended Assets</strong> API delivers ranked insights for the
  <strong> top 200 Bithumb-KRW assets</strong> by market cap, combining a six-dimensional scoring model
  with AI-generated Korean interpretation. The response contains <strong>all four sub-shapes</strong> as
  sibling keys at the top level — pick whichever shape your UI needs.
</p>

<Note>
  Two convenience variants return subsets of the same hourly snapshot:
  [`/v1/market/ai-buy-recommendation`](/ai-buy-recommendation) (rank with `total_score ≥ 6.0`, descending)
  and [`/v1/market/ai-sell-recommendation`](/ai-sell-recommendation) (`total_score < 4.0`, ascending).
</Note>

<h3 style={{ marginTop: 20 }}>The six-metric scoring model</h3>

Total Score = `Round(Average(6 metric scores), 1)`, range `0.0`–`10.0`. Each category is also `0.0`–`10.0` to one decimal place.

| Wire key | Korean (rendered) | English | Definition |
|---|---|---|---|
| `bullish_momentum` | 상승에너지 | Bullish Momentum | Price momentum |
| `trading_activity` | 거래활성도 | Trading Activity | Total trading volume |
| `search_trends` | 검색트렌드 | Search Trends | Public interest (search volume) |
| `price_stability` | 가격안정성 | Price Stability | Inverse of volatility (higher = more stable) |
| `market_sentiment` | 시장심리 | Market Sentiment | News + community + social media |
| `price_resilience` | 가격방어력 | Price Resilience | How well price holds support against drops |

<Warning>
  All score fields render with **exactly one decimal place** including trailing zero — `7.0`, `10.0`, `3.0`, never `7` / `10` / `3`. The Mintlify viewer may drop the trailing `.0` cosmetically; the wire bytes are correct.
</Warning>

<h3 style={{ marginTop: 20 }}>Score-to-opinion bands</h3>

`opinion` (and the band shown to users) is deterministically derived from `total_score`:

| `total_score` | `opinion` (Korean) | English |
|---|---|---|
| 0.0 – 1.9 | 매우 약세 | Very Weak |
| 2.0 – 3.9 | 약세 | Weak |
| 4.0 – 5.9 | 중립 | Neutral |
| 6.0 – 7.9 | 강세 | Strong |
| 8.0 – 10.0 | 매우 강세 | Excellent |

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Response Attributes</h2>

The top-level response is an object with **four sibling keys** — `score_ranking`, `summary`,
`comprehensive_evaluation`, `sentiment_trend` — each carrying the same envelope
(`{ uid, generated_at, assets: [...] }`) but with a different per-asset shape.

<ResponseField name="score_ranking" type="object" required>
  Full ranking with per-category ranks, opinion band, and Korean explanations per category.
  <Expandable title="envelope properties">
    <ResponseField name="uid" type="string" required>
      Document identifier for this batch.
    </ResponseField>
    <ResponseField name="generated_at" type="string" required>
      Snapshot wall-clock for this batch.
    </ResponseField>
    <ResponseField name="assets" type="array of objects" required>
      <Expandable title="per-asset properties">
        <ResponseField name="symbol" type="string" required>Bithumb exchange symbol (uppercase).</ResponseField>
        <ResponseField name="name" type="string" required>Korean asset name.</ResponseField>
        <ResponseField name="image_url" type="string" required>Asset icon URL.</ResponseField>
        <ResponseField name="uid" type="string" required>Per-asset document ID.</ResponseField>
        <ResponseField name="total_score" type="number" required>Overall score `0.0`–`10.0`, one decimal place.</ResponseField>
        <ResponseField name="score_by_category" type="object" required>
          Object with the six number keys `bullish_momentum / trading_activity / search_trends / price_stability /
          market_sentiment / price_resilience`. Each `0.0`–`10.0`, one decimal place. See the scoring model
          table above.
        </ResponseField>
        <ResponseField name="overall_rank" type="integer" required>Asset's rank by `total_score`, `1`–`200`.</ResponseField>
        <ResponseField name="ranking_by_category" type="object" required>
          Object with the same six keys as `score_by_category`. Each value is an integer `1`–`200` representing
          that asset's rank within the universe for that category.
        </ResponseField>
        <ResponseField name="opinion" type="string" required>
          Korean opinion band derived from `total_score`. One of `매우 약세 / 약세 / 중립 / 강세 / 매우 강세`.
        </ResponseField>
        <ResponseField name="interpretation" type="object" required>
          Object with the same six keys as `score_by_category`. Each value is a ~1-sentence Korean explanation
          (음/슴체) citing the data behind that category's score.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="summary" type="object" required>
  Same envelope as `score_ranking`. Each asset adds `title` and `summary` instead of the rank/opinion fields.
  <Expandable title="per-asset properties (in addition to common)">
    <ResponseField name="title" type="string" required>
      Korean headline derived from the `summary` body. **Max 20 Korean characters.** Generic boilerplate
      (e.g. `"검색흐름 양호, 전반 우세"`) is rejected; titles must capture the actual narrative.
    </ResponseField>
    <ResponseField name="summary" type="string" required>
      Multi-bullet Korean markdown synthesizing the 6 metric scores plus sentiment trend.
      Each bullet uses `\n- 헤드라인: 본문` form (per QA `ai_assets_1` § 1.5). The first reference to an
      asset includes the Korean name and bracketed symbol.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="comprehensive_evaluation" type="object" required>
  Same envelope (`{ uid, generated_at, assets: [...] }`) as `score_ranking`. Each asset carries the
  per-asset detail shape from SRS §4.6: the six `score_by_category` scores, `total_score`, `opinion`
  band, `title` (≤ 20 Korean characters, derived from `summary`), multi-bullet `summary`, and
  `content_creation_date` (`YYYY-MM-DD HH:mm` KST).
</ResponseField>

<ResponseField name="sentiment_trend" type="object" required>
  Same envelope as `score_ranking`. Filtered to tokens with sentiment data; sorted by bullish percentage
  descending.
  <Expandable title="per-asset properties (in addition to common)">
    <ResponseField name="bullish_pct" type="integer" required>
      Bullish percentage. Integer `0`–`100`.
    </ResponseField>
    <ResponseField name="bearish_pct" type="integer" required>
      Bearish percentage. Integer `0`–`100`. **Invariant: `bullish_pct + bearish_pct = 100`.**
    </ResponseField>
    <ResponseField name="sentiment_label" type="string" required>
      Korean keyword label derived from the bullish ratio. One of
      `매우 부정 / 부정 / 중립 / 긍정 / 매우 긍정`.
      The unspaced variants `매우부정` / `매우긍정` are tolerated on read; canonical output stays spaced.

      <Note>
        This sentiment surface is **distinct** from AI Report's
        [`community.overall_sentiment`](/ai-report#community), which uses a `{ score: 1–100, label }`
        struct. Don't unify the two.
      </Note>
    </ResponseField>
  </Expandable>
</ResponseField>
