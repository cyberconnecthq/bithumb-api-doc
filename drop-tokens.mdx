---
title: Today's Falling Coin
description: Korean news items on tokens with notable downside activity for the selected day, with per-article symbol, source, and KST publish time.
openapi: openapi.json GET /v1/market/drop-tokens
---

<div style={{ maxWidth: 760 }}>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>Today's Falling Coin</strong> API returns Korean news items focused on tokens showing notable
  downside activity for the selected day. Use the items as a daily "Top Losers" reading-list with rationale.
</p>

<Note>
  Powered by the daily Market Report pipeline (refresh **18:00 KST**, promotion completes by **17:50 KST**).
</Note>

<h3 style={{ marginTop: 20 }}>Query parameters</h3>

<ParamField query="date" type="string">
  Snapshot date in `YYYY-MM-DD` (KST). When omitted, the most recent snapshot is returned.
</ParamField>

<ParamField query="raw" type="boolean" default="false">
  When `true`, returns the **pre-polish** raw model output. Used by Bithumb QA; not for production traffic.
</ParamField>

---

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Response Attributes</h2>

The endpoint returns a JSON array of `TokenNewsItem` objects directly (no wrapping object).

<ResponseField name="[]" type="array of objects" required>
  News items relating to assets with notable downside, ordered by editorial priority. Each item:
  <Expandable title="item properties">
    <ResponseField name="title" type="string" required>
      Korean headline summary. Plain text, no Markdown.
    </ResponseField>
    <ResponseField name="content" type="string" required>
      Korean article excerpt explaining the rationale for the downside move. Sentence endings use 음/슴체.
    </ResponseField>
    <ResponseField name="source" type="string" required>
      Publisher name (Korean or English as published).
    </ResponseField>
    <ResponseField name="datetime" type="string" required>
      Article publish time in **Asia/Seoul (KST)**. Wire format: `YYYY-MM-DD HH:MM:SS`.
    </ResponseField>
    <ResponseField name="publish_time" type="string" required>
      Snapshot publish time (when this curated list was promoted), KST `YYYY-MM-DD HH:MM:SS`. Same value
      across all items in a single response.
    </ResponseField>
  </Expandable>
</ResponseField>

---

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Errors</h2>

| Status | Code | When |
|---|---|---|
| <span style={{color:"#d97706",fontWeight:600}}>● 400</span> | `INVALID_PARAMS` | `date` is not `YYYY-MM-DD`. |
| <span style={{color:"#d97706",fontWeight:600}}>● 404</span> | `NOT_FOUND` | No snapshot exists for `date`. |
| <span style={{color:"#dc2626",fontWeight:600}}>● 500</span> | `INTERNAL_ERROR` | Unexpected upstream failure. |

</div>
