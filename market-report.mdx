<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>Market Report</strong> API returns a point-in-time snapshot that aggregates
  <strong> six independent market views</strong> into one response: AI-highlighted news, today's rising and
  falling tokens, market sentiment, key trend events, and major events tonight.
</p>

<Note>
  Each section comes from its own upstream batch and surfaces its own `publish_time`; sections do not
  share a batch timestamp. Missing sections render as `null` or empty arrays rather than failing the
  whole response.
</Note>

<h3 style={{ marginTop: 20 }}>Cadence & freshness</h3>

<CardGroup cols={2}>
  <Card title="Refresh" icon="clock">
    Daily at **18:00 KST**. Promotion to public read tables completes by **17:50 KST**.
  </Card>
  <Card title="Surface scope" icon="newspaper">
    **Text + ranked lists only**. Charts are scoped to the AI Report surface only.
  </Card>
</CardGroup>

<h3 style={{ marginTop: 20 }}>Cross-cutting conventions</h3>

<AccordionGroup>
  <Accordion title="Language, currency, timezone">
    All user-facing strings are **Korean**. Currency is **KRW (`원`)** with `만 / 억 / 조` magnitude
    abbreviations. Timestamps are **Asia/Seoul (KST)**. UTC and USD are forbidden in user-facing fields.
  </Accordion>

  <Accordion title="Sector taxonomy (closed 14-sector enum)">
    When sectors appear (e.g. inside `key_trend_events` or news content), they are restricted to:
    `AI · DeFi · DePIN · Fan Token · Gaming · Layer1 · Layer2 · Meme · Metaverse · NFT · Payments · RWA · Service · Social`.
    Anything outside this set is invalid.
  </Accordion>

  <Accordion title="Tone — 음/슴체">
    Korean prose uses 음/슴체 sentence endings (`~함`, `~판단됨`, `~예상됨`). Banned: `입니다`, `습니다`,
    `합니다`, `해요`, `있습니다`, `~다.`, `~입니다.`.
  </Accordion>
</AccordionGroup>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Response Attributes</h2>

The top-level response contains **six sibling keys**, each corresponding to one market view.

<ResponseField name="market_sentiment" type="object" required>
  Fear & Greed index, breadth counts (Up / Flat / Down), and gainer count vs previous day.
  <Expandable title="properties">
    <ResponseField name="report_time" type="string" required>
      Reference time the snapshot represents (KST). Sample-data validation runs use canonical
      `YYYY-MM-DD 07:50` (per SRS §5.1).
    </ResponseField>
    <ResponseField name="publish_time" type="string" required>
      Wall-clock when this section was promoted (KST).
    </ResponseField>
    <ResponseField name="fear_greed_index" type="integer" required>
      Fear & Greed score. Range: `0`–`100`. Banded into 5 categories: Extreme Fear / Fear / Neutral /
      Greed / Extreme Greed.
    </ResponseField>
    <ResponseField name="fear_greed_interpretation" type="string" required>
      Korean prose interpretation of the index.
    </ResponseField>
    <ResponseField name="rising_assets_vs_previous_day" type="integer" required>
      Count of Bithumb-KRW assets whose 24-hour change is positive vs the prior day's snapshot.
    </ResponseField>
    <ResponseField name="market_price_fluctuations" type="array of objects" required>
      Breadth counts — typically 3 entries (Up / Flat / Down). Each item: `{ type: "UP" | "FLAT" | "DOWN", num: integer }`.
      Sum across the three entries equals the size of the Bithumb-KRW market at `report_time`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="must_know_news" type="array of objects" required>
  AI-highlighted macro/market news for quick daily context. Korean only. Each item:
  `{ title, content, source, datetime, publish_time }` (KST timestamps).
</ResponseField>

<ResponseField name="rise_tokens" type="array of objects" required>
  News items on Bithumb-listed assets with notable upside, with rationale. Same item shape as
  `must_know_news`.
</ResponseField>

<ResponseField name="drop_tokens" type="array of objects" required>
  News items on Bithumb-listed assets with notable downside, with rationale. Same item shape as
  `must_know_news`.
</ResponseField>

<ResponseField name="key_trend_events" type="array of arrays of objects" required>
  **2-D structure.** Outer array contains multiple "versions" of the trend lineup; inner array is a
  sequence of trend events for that version.

  Each inner item: `{ title, summary }` — Korean strings.
</ResponseField>

<ResponseField name="market_events" type="object" required>
  Major upcoming KST-scheduled events.
  <Expandable title="properties">
    <ResponseField name="datetime" type="string" required>Snapshot reference time (KST).</ResponseField>
    <ResponseField name="publish_time" type="string" required>Promotion wall-clock (KST).</ResponseField>
    <ResponseField name="major_events" type="array of objects" required>
      Minimum **5 events** by importance level (per SRS §5.3.3). Each item: `{ event_name, datetime }`.
      Limited to **Korean and US events** (FOMC, interest-rate decisions, US/Korea earnings calendar, etc.).
    </ResponseField>
  </Expandable>
</ResponseField>
