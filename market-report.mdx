<div style={{ maxWidth: 760 }}>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>Market Report</strong> API returns a point-in-time snapshot that aggregates
  <strong> six independent market views</strong> into one response: AI-highlighted news, today's rising and
  falling tokens, market sentiment, key trend events, and major events tonight.
</p>

<Note>
  Each section comes from its own upstream batch and surfaces its own `publish_time`; sections do not
  share a batch timestamp. Missing sections render as `null` or empty arrays rather than failing the
  whole response.
</Note>

<h3 style={{ marginTop: 20 }}>Cadence & freshness</h3>

<CardGroup cols={2}>
  <Card title="Refresh" icon="clock">
    Daily at **18:00 KST** (per SRS §5.1). Promotion to public read tables completes by **17:50 KST**.
  </Card>
  <Card title="Surface scope" icon="newspaper">
    **Text + ranked lists only**. Charts are scoped to the AI Report surface only.
  </Card>
</CardGroup>

---

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Response Attributes</h2>

The endpoint returns the standard envelope `{ success, message, data }`. The `data` object contains six
sibling sections (field order below matches the wire sample):

<ResponseField name="must_know_news" type="array of objects" required>
  AI-highlighted macro/market news for quick daily context. Korean only.
  <Expandable title="item properties (in wire order)">
    <ResponseField name="title" type="string" required>Korean headline.</ResponseField>
    <ResponseField name="content" type="string" required>Korean article excerpt / supporting context.</ResponseField>
    <ResponseField name="source" type="string" required>
      Source attribution. Wire sample shows real publisher names (e.g. `"COINREADERS"`) and internal
      labels (e.g. `"업계 소식통"`).
    </ResponseField>
    <ResponseField name="datetime" type="string" required>
      Article publish time, KST. Format: `YYYY-MM-DD HH:MM`.
    </ResponseField>
    <ResponseField name="publish_time" type="string" required>
      Section promotion wall-clock, KST. Format: `YYYY-MM-DD HH:MM`. Same value across all items in this
      section.
    </ResponseField>
    <ResponseField name="symbol" type="string" required>
      Asset symbol the news item is associated with.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="rise-tokens" type="array of objects" required>
  News items on Bithumb-listed assets with notable upside, with rationale.

  <Note>
    Wire key uses **hyphen**: `rise-tokens` (not `rise_tokens`).
  </Note>

  Item shape is identical to `must_know_news`: `{ title, content, source, datetime, publish_time, symbol }`.
</ResponseField>

<ResponseField name="drop-tokens" type="array of objects" required>
  News items on Bithumb-listed assets with notable downside, with rationale.

  <Note>
    Wire key uses **hyphen**: `drop-tokens` (not `drop_tokens`).
  </Note>

  Item shape is identical to `must_know_news`: `{ title, content, source, datetime, publish_time, symbol }`.
</ResponseField>

<ResponseField name="market_sentiment" type="object" required>
  Fear & Greed index, breadth counts (Up / Flat / Down), and gainer count vs previous day.
  <Expandable title="properties (in wire order)">
    <ResponseField name="fear_greed_index" type="integer" required>
      Fear & Greed score. Range: `0`–`100`. Banded into 5 categories per SRS §5.3.1: Extreme Fear / Fear /
      Neutral / Greed / Extreme Greed.
    </ResponseField>
    <ResponseField name="fear_greed_interpretation" type="string" required>
      Korean keyword label for the band. Wire sample shows the bare keyword (e.g. `"중립"`), not a
      sentence.
    </ResponseField>
    <ResponseField name="market_price_fluctuations" type="array of objects" required>
      Breadth counts — typically 3 entries (Up / Flat / Down). Each item:
      `{ type: "UP" | "FLAT" | "DOWN", num: integer }`.
    </ResponseField>
    <ResponseField name="rising_assets_vs_previous_day" type="integer" required>
      **Delta** in the count of rising assets vs the previous day's snapshot (signed integer). Wire
      sample shows `2` (not the absolute count of rising assets).
    </ResponseField>
    <ResponseField name="fear_greed_vs_previous_day" type="integer" required>
      Change in `fear_greed_index` vs the previous day's snapshot (signed integer).
    </ResponseField>
    <ResponseField name="report_time" type="string" required>
      Reference time the snapshot represents, KST. Wire sample uses **dot-separated** date format:
      `YYYY.MM.DD HH:MM`.
    </ResponseField>
    <ResponseField name="publish_time" type="string" required>
      Section promotion wall-clock, KST. Wire sample uses **hyphen-separated** date format:
      `YYYY-MM-DD HH:MM`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="key_trend_events" type="array of arrays of objects" required>
  **2-D structure** (outer array of arrays). Wire sample contains a single outer entry with multiple
  inner trend events.

  Each inner item: `{ title, summary }` — Korean strings.
</ResponseField>

<ResponseField name="market_events" type="object" required>
  Major upcoming KST-scheduled events.
  <Expandable title="properties (in wire order)">
    <ResponseField name="major_events" type="array of objects" required>
      Each item: `{ event_name, datetime }` (KST `YYYY-MM-DD HH:MM`). SRS §5.3.3 requires minimum 5
      events; wire sample observed with 3. Limited to **Korean and US events** (FOMC, interest-rate
      decisions, US/Korea earnings calendar, etc.).
    </ResponseField>
    <ResponseField name="publish_time" type="string" required>
      Section promotion wall-clock, KST. Format: `YYYY-MM-DD HH:MM`.
    </ResponseField>
    <ResponseField name="event_influence" type="string" required>
      Korean string. Wire sample shows a placeholder value (`"이벤트 영향도"`); semantic content not
      yet confirmed.
    </ResponseField>
  </Expandable>
</ResponseField>

</div>

<RequestExample>

```bash cURL
curl --request GET \
  --url "https://api.asksurf.ai/surf-ai/v1/market/report" \
  --header "Authorization: <api-key>"
```

```python Python
import requests

response = requests.get(
    "https://api.asksurf.ai/surf-ai/v1/market/report",
    headers={"Authorization": "<api-key>"},
)
report = response.json()
```

```javascript JavaScript
const response = await fetch(
  "https://api.asksurf.ai/surf-ai/v1/market/report",
  { headers: { Authorization: "<api-key>" } },
);
const report = await response.json();
```
