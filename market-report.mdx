<div style={{ maxWidth: 760 }}>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>Market Report</strong> API returns a point-in-time snapshot that aggregates
  <strong> six independent market views</strong> into one response: AI-highlighted news, today's rising and
  falling tokens, market sentiment, key trend events, and major events tonight.
</p>

<Note>
  Each section comes from its own upstream batch and surfaces its own `publish_time`; sections do not
  share a batch timestamp. Missing sections render as `null` or empty arrays rather than failing the
  whole response.
</Note>

<h3 style={{ marginTop: 20 }}>Cadence & freshness</h3>

<CardGroup cols={2}>
  <Card title="Refresh" icon="clock">
    Daily at **07:50 KST**.
  </Card>
  <Card title="Surface scope" icon="newspaper">
    **Text + ranked lists only**. Charts are scoped to the AI Report surface only.
  </Card>
</CardGroup>

---

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Response Attributes</h2>

The endpoint returns the standard envelope `{ success, message, data }`. The `data` object contains six
sibling sections (field order below matches the wire sample).

`must_know_news`, `rise-tokens`, and `drop-tokens` are all **Key News** sections (SRS §5.3.2).
`rise-tokens` and `drop-tokens` share the same item shape and both include a `symbol` field.
`must_know_news` (AI-Featured Articles) has no `symbol`.

<ResponseField name="must_know_news" type="array of objects" required>
  AI-Featured Articles — macro/market news highlighted by AI. Korean only.
  <Expandable title="item properties">
    <ResponseField name="title" type="string" required>Korean news headline summary.</ResponseField>
    <ResponseField name="source" type="string" required>
      Article source — media company name or Twitter/X account name.
    </ResponseField>
    <ResponseField name="datetime" type="string" required>
      Article publish time, KST. Format: `YYYY-MM-DD HH:MM`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="rise-tokens" type="array of objects" required>
  Top Gaining Coins Today — Bithumb-listed assets with notable upside, with rationale.
  <Expandable title="item properties">
    <ResponseField name="symbol" type="string" required>Asset symbol.</ResponseField>
    <ResponseField name="title" type="string" required>Korean news headline summary.</ResponseField>
    <ResponseField name="source" type="string" required>
      Article source — media company name or Twitter/X account name.
    </ResponseField>
    <ResponseField name="datetime" type="string" required>
      Article publish time, KST. Format: `YYYY-MM-DD HH:MM`.
    </ResponseField>
  </Expandable>

  <Note>
    Wire key uses **hyphen**: `rise-tokens` (not `rise_tokens`).
  </Note>
</ResponseField>

<ResponseField name="drop-tokens" type="array of objects" required>
  Top Losing Coins Today — Bithumb-listed assets with notable downside, with rationale.
  Same item shape as `rise-tokens` above.

  <Note>
    Wire key uses **hyphen**: `drop-tokens` (not `drop_tokens`).
  </Note>
</ResponseField>

<ResponseField name="market_sentiment" type="object" required>
  Fear & Greed index, breadth counts (Up / Flat / Down), and gainer count vs previous day.
  <Expandable title="properties (in wire order)">
    <ResponseField name="fear_greed_index" type="integer" required>
      Fear & Greed score. Range: `0`–`100`. Banded into 5 categories: Extreme Fear / Fear / Neutral /
      Greed / Extreme Greed.
    </ResponseField>
    <ResponseField name="market_price_fluctuations" type="array of objects" required>
      Up / Flat / Down counts for all coins on the Bithumb KRW market. Each item:
      `{ type: "UP" | "FLAT" | "DOWN", num: integer }`.
    </ResponseField>
    <ResponseField name="rising_assets_vs_previous_day" type="integer" required>
      Number of gainers vs the previous day (signed integer).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="key_trend_events" type="array of arrays of objects" required>
  Key macroeconomic and market trend events. 2-D structure — outer array of groups, each group an array
  of trend items. Each item: `{ title, summary }` — Korean strings.
</ResponseField>

<ResponseField name="market_events" type="object" required>
  Major upcoming KST-scheduled events.
  <Expandable title="properties (in wire order)">
    <ResponseField name="major_events" type="array of objects" required>
      Each item: `{ event_name, datetime }` (KST `YYYY-MM-DD HH:MM`). SRS §5.3.3 requires minimum 5
      events; wire sample observed with 3. Limited to **Korean and US events** (FOMC, interest-rate
      decisions, US/Korea earnings calendar, etc.).
    </ResponseField>
    <ResponseField name="publish_time" type="string" required>
      Section promotion wall-clock, KST. Format: `YYYY-MM-DD HH:MM`.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash cURL
curl --request GET \
  --url "https://api.asksurf.ai/surf-ai/v1/market/report" \
  --header "Authorization: <api-key>"
```

```python Python
import requests

response = requests.get(
    "https://api.asksurf.ai/surf-ai/v1/market/report",
    headers={"Authorization": "<api-key>"},
)
report = response.json()
```

```javascript JavaScript
const response = await fetch(
  "https://api.asksurf.ai/surf-ai/v1/market/report",
  { headers: { Authorization: "<api-key>" } },
);
const report = await response.json();
```
